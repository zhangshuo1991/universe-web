# .impeccable.md — Universe Web Design Context

## Design Context

### Users

面向**普通公众**的地球观测站。用户是对太空、地球科学和地球实时状态感兴趣的普通访客，不默认具备专业背景，也不应被复杂术语和密集控制面板劝退。他们来到这里，是为了用最低认知负担去“看看现在的地球发生了什么”，并在探索过程中自然理解卫星轨道、天气、地震、空间天气等信息。使用场景通常是桌面浏览器上的沉浸式探索，也需要兼顾移动端的基本可用性。

### Brand Personality

**探索 · 智能 · 简洁**

- **探索 (Exploratory)**: 界面应像太空望远镜的取景器——每一次交互都在揭示新发现。鼓励好奇心，让用户感到“再点一下就能看到更多”。
- **智能 (Intelligent)**: AI 对话和数据可视化传递“懂你想问什么”的感觉。信息呈现应精确而不堆砌，给人值得信赖的专业印象。
- **简洁 (Simple)**: UI 必须简单、好用。用户不该先学习界面，再开始探索地球。层级、文案、操作路径都应足够直观，让第一次进入的人也能立即上手。

语调：沉稳、清晰、不卖弄，像一位知识丰富但表达通俗的天文馆解说员。

### Aesthetic Direction

**视觉基调**: 深空科技感，但必须服务于可用性。保留纯暗色主题、玻璃态面板、微妙的光晕与网格纹理，但不能为了“酷”牺牲信息理解和操作效率。

**参考方向**:
- 当前没有固定外部参考产品。
- 所有视觉决策应优先围绕“普通公众一眼能懂、第一次用就会、愿意继续探索”来判断，而不是围绕设计潮流或技术炫技。

**反面参考** (应避免的方向):
- 过度拟物或游戏化的卡通风格
- 信息过载的传统仪表盘 (满屏数字、控件和状态，缺少主次)
- 过于极客/黑客风 (纯 monospace 终端感)
- 把“专业感”误做成“复杂难懂”的数据后台

**主题**: 仅暗色模式 (深空背景 `#050913`)

**色彩系统** (已建立):
| Token | 值 | 用途 |
|-------|------|------|
| `--bg` | `#050913` | 主背景 |
| `--bg-secondary` | `#091324` | 次级背景 |
| `--panel` | `rgba(8,18,32,0.9)` | 面板背景 |
| `--text` | `#edf4ff` | 主文本 (冷白) |
| `--muted` | `#96aeca` | 次级文本 (蓝灰) |
| `--aqua` | `#5eead4` | 主重点色 (青绿) |
| `--warm` | `#f7b955` | 辅助重点色 (琥珀金) |
| `--danger` | `#f87171` | 警告/错误 |

**字体组合**:
- 正文: Literata (衬线) — 传递文化感与可读性
- UI/标题: Oxanium (几何无衬线) — 传递科技感
- CJK 回退: PingFang SC → Hiragino Sans GB → Noto Sans CJK SC

**空间与形状**:
- 圆角: 18-32px (大圆角，柔和科技感)
- 间距: 基于 18px 的间距系统
- 面板: 玻璃态 (半透明 + backdrop-blur + 内发光边框)
- 布局: CSS Grid，三栏 (sidebar 104px / viewer / detail 360px)

### Design Principles

1. **简单先于丰富 (Simplicity Before Density)** — 第一原则不是“放下更多信息”，而是“让普通公众立即会用”。任何面板、卡片、按钮、图层入口，都应先回答“它是否足够直观”，再考虑是否继续加内容。

2. **数据即叙事 (Data as Story)** — 每一项数据都应有上下文和意义，而非孤立的数字。卫星数量不如“正在追踪 X 颗卫星中最亮的 ISS”有故事感。让信息自己讲述地球的故事。

3. **渐进式揭示 (Progressive Disclosure)** — 首屏保持干净壮美，细节在用户表达兴趣时展开。不要一次性铺满所有数据层。用分层、折叠、AI 对话来引导深度探索。

4. **探索欲优先 (Design for Curiosity)** — 界面要让用户自然产生“我想继续点、继续看、继续问”的冲动。设计不是把信息塞满，而是把入口做得足够轻、顺、明确，让探索本身变得顺手。

5. **空间感优先 (Spatial Hierarchy)** — 地球 viewer 永远是视觉焦点。面板是它的配角，半透明、不抢占注意力。信息层次靠亮度、透明度和排版节奏区分，而非堆砌色块和边框。

6. **无障碍 AA 基线 (WCAG AA Baseline)** — 继续以 WCAG AA 作为默认基线。所有文本对比度 ≥ 4.5:1 (小字) / 3:1 (大字)。交互元素有清晰的 focus 状态 (`--aqua` 轮廓)。动效尊重 `prefers-reduced-motion`。信息不仅靠颜色传达。
